# MPU6050

6-осевой IMU: 3-осевой акселерометр + 3-осевой гироскоп + датчик температуры.
Интерфейс: I2C, до 400 кГц (Fast Mode).

## Распиновка (опора: F407VG-DISC1 + I2C1)

| Контакт MPU6050 | Пин STM32 | Примечание |
|-----------------|-----------|------------|
| VCC | 3.3 V | |
| GND | GND | |
| SCL | PB6 | I2C1 SCL, AF4, open-drain, pull-up |
| SDA | PB7 | I2C1 SDA, AF4, open-drain, pull-up |
| AD0 | GND | Выбирает 7-битный адрес `0x68` (или `0x69` при подтяжке к VCC) |
| INT | не подключен | Опционально, драйвером не используется |

I2C-адреса (7 бит): `0x68` (AD0 = GND), `0x69` (AD0 = VCC).

## Драйвер

```cpp
import driver.stm32f4.i2c;
import sensor.mpu6050;

using sensor::Mpu6050;

driver::stm32f4::I2c g_i2c1{*I2C1, {.clockSpeed = 400000, .fastMode = true}};
Mpu6050 g_imu{g_i2c1, {.address = 0x68}};
```

### Сэмплирование

```cpp
g_imu.init();

Mpu6050::Sample s;
if (g_imu.read(s) == driver::Status::Ok) {
    // s.ax, s.ay, s.az  (int16, raw counts ±32767 для ±2g)
    // s.gx, s.gy, s.gz  (int16, raw counts)
    // s.temp            (int16, raw; °C = temp/340 + 36.53)
}
```

Драйвер читает все 14 байт (accel + temp + gyro) одной I2C-транзакцией —
сэмплы во времени когерентны.

## Ожидаемый serial-выход (шаблон `mpu6050-uart`)

```
A: 123 -45 16384  G: 12 -3 5  T: 23
A: 122 -44 16380  G: 11 -2 4  T: 23
...
```

В покое (модуль на плоской поверхности): `|a| ≈ 16384` (1 g для диапазона ±2g),
ось Z доминирует.

## Верификация

После прошивки `freertos/mpu6050-uart`:

1. Открыть FTDI на `/dev/tty.usbserial-*` (macOS) или `/dev/ttyUSB0` (Linux),
   115200 baud.
2. Сэмплы приходят ~10 Гц.
3. `|a|` в диапазоне `[15500, 16800]` (1 g ±~5%).
4. Шум гироскопа `≤ ±5` единиц в покое.
5. Температура около комнатной (≈ 23 °C), монотонна.
