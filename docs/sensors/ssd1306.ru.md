# OLED-дисплей SSD1306

Монохромный OLED-контроллер 128×64. Драйвер заточен под I2C-вариант;
SPI-вариант пока не реализован.

## Распиновка

| Контакт OLED | Пин STM32 | Примечание |
|--------------|-----------|------------|
| VCC | 3.3 V | Некоторые модули принимают 5 V — проверьте шелкографию |
| GND | GND | |
| SCL | PB6 | I2C1 SCL, AF4 |
| SDA | PB7 | I2C1 SDA, AF4 |

I2C-адрес (7 бит): чаще всего `0x3C` (запись 8-бит `0x78`, чтение `0x79`).
Иногда модули используют `0x3D` (`0x7A` / `0x7B`). Если шелкография
не читается — запустите шаблон
[`bare-metal/i2c-scan`](https://github.com/khosta77/stm32-sdk/tree/develop/templates/bare-metal/i2c-scan),
он печатает все адреса на шине.

## Даташит

- [SSD1306 datasheet (Solomon Systech)](https://cdn-shop.adafruit.com/datasheets/SSD1306.pdf)

## Драйвер

```cpp
import driver.stm32f4.i2c;
import sensor.ssd1306;

using sensor::Ssd1306;

driver::stm32f4::I2c g_i2c1{*I2C1, {.clockSpeed = 400000, .fastMode = true}};

Ssd1306 g_oled{
    g_i2c1,
    {
        .addr = 0x3C,
        .contrast = 0x7F,
        .flipH = false,
        .flipV = false,
    },
};
```

### Рисование

```cpp
g_oled.init();
g_oled.clear();
g_oled.drawText(0, 0, "STM32 SDK");
g_oled.drawText(0, 12, "SSD1306 128x64");
g_oled.flush();
```

Драйвер держит framebuffer 1024 байт (128 × 64 / 8) в RAM. `flush()` отсылает
весь framebuffer по I2C — на 400 кГц это ~25 мс.

### Шрифт

Встроенный ASCII-шрифт 5×7 (0x20–0x7E), 95 глифов, column-major LSB-first. Для
крупного текста или anti-aliasing — рендерите off-line и копируйте во
framebuffer через `setPixel`.

## Верификация

После прошивки `freertos/oled-display-test`:

1. Подайте питание; в течение ~200 мс дисплей зажигается.
2. Сверху появляется заголовок `STM32 SDK`.
3. Строка `uptime: Ns` обновляется раз в секунду.
4. Без тиринга и битых пикселей.
