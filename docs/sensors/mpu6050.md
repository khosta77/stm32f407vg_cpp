# MPU6050

6-axis IMU: 3-axis accelerometer + 3-axis gyroscope + temperature sensor.
Interface: I2C, up to 400 kHz (Fast Mode).

## Pinout (reference: F407VG-DISC1 + I2C1)

| MPU6050 pin | STM32 pin | Notes |
|-------------|-----------|-------|
| VCC | 3.3 V | |
| GND | GND | |
| SCL | PB6 | I2C1 SCL, AF4, open-drain, pull-up |
| SDA | PB7 | I2C1 SDA, AF4, open-drain, pull-up |
| AD0 | GND | Selects 7-bit address 0x68 (vs 0x69 if pulled high) |
| INT | unused | Optional, not used by the driver |

I2C addresses (7-bit): `0x68` (AD0 = GND), `0x69` (AD0 = VCC).

## Driver

```cpp
import driver.stm32f4.i2c;
import sensor.mpu6050;

using sensor::Mpu6050;

driver::stm32f4::I2c g_i2c1{*I2C1, {.clockSpeed = 400000, .fastMode = true}};
Mpu6050 g_imu{g_i2c1, {.address = 0x68}};
```

### Sampling

```cpp
g_imu.init();

Mpu6050::Sample s;
if (g_imu.read(s) == driver::Status::Ok) {
    // s.ax, s.ay, s.az  (int16, raw counts ±32767 for ±2g)
    // s.gx, s.gy, s.gz  (int16, raw counts)
    // s.temp            (int16, raw; °C = temp/340 + 36.53)
}
```

The driver reads all 14 bytes (accel + temp + gyro) in one I2C transaction
to ensure samples are temporally coherent.

## Expected serial output (in the `mpu6050-uart` template)

```
A: 123 -45 16384  G: 12 -3 5  T: 23
A: 122 -44 16380  G: 11 -2 4  T: 23
...
```

Resting on a flat surface, `|a| ≈ 16384` (1 g at ±2g range), with Z dominant.

## Verification

After flashing `freertos/mpu6050-uart`:

1. Open FTDI on `/dev/tty.usbserial-*` (macOS) or `/dev/ttyUSB0` (Linux) at 115200 baud.
2. Confirm samples arrive at ~10 Hz.
3. `|a|` should be in `[15500, 16800]` (1 g ±~5%).
4. Gyro noise should be `≤ ±5` counts on resting module.
5. Temperature monotonically near room temperature (≈ 23 °C).
