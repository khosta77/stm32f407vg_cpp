# SSD1306 OLED display

Monochrome 128×64 OLED controller. The driver targets the I2C variant; SPI
variant is not currently implemented.

## Pinout

| OLED pin | STM32 pin | Notes |
|----------|-----------|-------|
| VCC | 3.3 V | Some modules accept 5 V — check the silkscreen |
| GND | GND | |
| SCL | PB6 | I2C1 SCL, AF4 |
| SDA | PB7 | I2C1 SDA, AF4 |

I2C address (7-bit): `0x3C` (8-bit write `0x78`, read `0x79`) is the most common.
Some modules use `0x3D` (`0x7A` / `0x7B`). If your silkscreen is unreadable, run
the [`bare-metal/i2c-scan`](https://github.com/khosta77/stm32-sdk/tree/develop/templates/bare-metal/i2c-scan)
template — it prints every address on the bus.

## Driver

```cpp
import driver.stm32f4.i2c;
import sensor.ssd1306;

using sensor::Ssd1306;

driver::stm32f4::I2c g_i2c1{*I2C1, {.clockSpeed = 400000, .fastMode = true}};

Ssd1306 g_oled{
    g_i2c1,
    {
        .addr = 0x3C,
        .contrast = 0x7F,
        .flipH = false,
        .flipV = false,
    },
};
```

### Drawing

```cpp
g_oled.init();
g_oled.clear();
g_oled.drawText(0, 0, "STM32 SDK");
g_oled.drawText(0, 12, "SSD1306 128x64");
g_oled.flush();
```

The driver holds a 1024-byte framebuffer (128 × 64 / 8) in RAM. `flush()` sends
the entire framebuffer over I2C — at 400 kHz this takes ~25 ms.

### Font

Built-in 5×7 column-major LSB-first ASCII font (0x20–0x7E), 95 glyphs. For
larger text or anti-aliasing, render off-line and copy into the framebuffer via
`setPixel`.

## Verification

After flashing `freertos/oled-display-test`:

1. Power up the board; within ~200 ms the display should light up.
2. Title text `STM32 SDK` appears at the top.
3. A line `uptime: Ns` updates once per second.
4. No tearing or stuck pixels.
